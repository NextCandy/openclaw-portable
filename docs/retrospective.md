## 💡 学到的经验

### 1. GitHub 开发标准
- 📝 **必须先创建 issue** - 即使是自己提的
- 📝 **必须写任务卡** - 明确目标、边界、非目标
- 📝 **必须列改动点** - 记录所有改动
- 📝 **必须完成 4 层验证** - 语法、导入、行为、回归
- 📝 **必须审查 diff** - 检查改动量、非目标区域
- 📝 **必须写完整发布说明** - 问题、修复、验证、非修改
- 📝 **必须复盘** - 总结经验教训

### 2. Git 使用
- 📝 **大文件用 Git LFS** - 超过 100MB 的文件
- 📝 **统一分支** - 只用 main 或 master，不要两者都有
- 📝 **.gitignore 要完善** - 忽略大文件和临时文件

### 3. 便携版开发
- 📝 **预置依赖** - 避免用户下载
- 📝 **清理测试文件** - 减少体积
- 📝 **跨平台支持** - Windows (WSL2) + Linux/macOS
- 📝 **友好提示** - 启动/关闭时的友好消息
